> 日期：2026-07-20

## 已通过范围

- 精确小数解析与每赛项分数范围。
- 固定按“赛区＋赛项＋组别”分区。
- 标准竞赛排名 `1、2、2、4`，同分保持稳定来源顺序。
- 无效成绩排除并给出具体异常原因。
- SQLite 原子快照、内容去重、失败回滚和历史恢复。
- 手机号/身份证号列加密和大屏脱敏 DTO。
- 表格粘贴、CSV、XLSX 的候选预览与哈希确认发布。
- 当前分组、当前赛项、全部赛项 Excel/ZIP 导出。
- iOS 风格后台、深色脱敏大屏和新鲜度提示。
- Google Chrome 端到端流程：后台排名与异常、大屏脱敏、手动预览并发布、可追溯分组导出、键盘焦点和全路由 WCAG AA 自动扫描。
- 持久化同步运行记录、可调新鲜度阈值、快照查看/比较/恢复，以及恢复后按内容哈希复用既有历史版本。
- 大屏对超过 12 条的分区分页，并在自动轮播时覆盖全部页和排名分区。

## 规模与性能

自动性能测试分别对 501 条和 5,000 条结构等价合成记录执行“校验排名＋SQLite 内存快照提交”，每组目标均为 1 秒以内。5,000 条用例已在本机通过，具体时间随机器负载变化，硬性判断以 `< 1,000 ms` 断言为准。另有 5,000 条分赛项 Excel 导出测试；最近一次完整质量门约 609 ms，硬性上限为 10 秒。

性能数据不包含金山页面加载、人工复制、文件选择或浏览器下载时间。

## 4 小时稳定性

仓库提供：

```bash
pnpm build
pnpm test:stability
pnpm test:stability:ui
```

默认持续 240 分钟，每 30 秒检查状态、排名、大屏 DTO、快照数量和服务 RSS：

- 当前快照 ID 不漂移；
- 排名 API 与大屏脱敏 DTO 始终为 501 条；生产 DOM 按设计保持后台摘要 8 条、大屏单页 1–12 条；
- 没有重复快照；
- 服务状态保持 `synced`；
- 预热后的 API 与 UI 服务持续/滚动窗口内存增长不超过 128 MiB，系统 Chrome 不超过 384 MiB；完整峰谷范围只作诊断记录。
- 单次本机连接失败最多重试两次并记录；全程累计超过 3 次仍判定失败。

第一条测试持续检查后台排名和展示 DTO；第二条会启动生产构建、系统 Google Chrome、管理后台和大屏，持续核对真实 DOM、过期提示、服务内存和浏览器内存。机器可读结果分别保存到 `logs/stability-report.json` 和 `logs/ui-stability-report.json`；该目录被 Git 忽略，只包含合成记录计数、快照 ID、采样时间和内存数值。

开发时可用 `STABILITY_MINUTES=1 pnpm test:stability` 做脚本冒烟，但这不能替代 4 小时验收。第一轮正式 API 长稳测试在结束前因一次本机连接 `ECONNRESET` 失败，不能计为通过。

第二轮 API/UI 正式测试从 2026-07-19 23:01:10（北京时间）运行到 2026-07-20 03:01:12，均跑满 240 分钟，但仍不能计为通过：API 报告含 399 个成功样本、12 次瞬时请求失败，最大采样空档 1,037.14 秒；UI 报告含 402 个成功样本、6 次瞬时请求失败，最大采样空档 1,015.42 秒。所有成功样本均保持 501 条、单一快照和正确 DOM，但空档证明 Mac 曾睡眠/唤醒，未形成连续 4 小时证据。UI 另因旧算法把 Chrome 启动峰值 870.94 MiB 到后续最低值 206.20 MiB 的回落范围误记为“增长 664.74 MiB”；预热后首个稳定窗口中位数为 383.09 MiB，末窗口为 283.81 MiB，实际没有持续上升证据。

此后监测器增加了 macOS 唤醒保持、采样空档检查、请求重试明细和预热后稳定窗口内存算法，并要求从零执行第三轮完整 240 分钟。第二轮失败不计入通过时长。

修正后的监测器已用当前生产构建完成短时双通道烟测：API 与 UI 各采集 6 个样本，均保持 501 条、1 个快照和 0 次瞬时请求失败；最大采样间隔分别为 1.02 秒和 1.04 秒，门槛为 2 秒，并确认报告包含 `caffeinate-ims` 唤醒锁。API 稳定窗口增长 3.44 MiB、UI 服务 4.54 MiB；系统 Chrome 从启动窗口 833.38 MiB 回落到 810.05 MiB，持续增长为 0。该结果只证明修正后的监测器和生产路径可工作，不替代正式 240 分钟验收。

第三轮正式 API/UI 长稳测试从 2026-07-20 03:13:27（北京时间）运行到 2026-07-20 07:13:29，并由 `caffeinate-ims` 全程保持系统唤醒。两条通道均通过：

| 指标 | API 通道 | 生产 UI + Google Chrome 通道 |
|---|---:|---:|
| 连续时长 | 240 分钟 | 240 分钟 |
| 有效样本 | 480（门槛 456） | 480（门槛 456） |
| 最大采样间隔 | 30.49 秒（门槛 60 秒） | 30.10 秒（门槛 60 秒） |
| 瞬时请求失败 | 0 | 0 |
| 记录数 / 快照数 | 501 / 1 | 501 / 1 |
| 服务内存基线 → 末窗口 | 74.88 → 40.19 MiB | 83.87 → 48.22 MiB |
| 服务持续增长 / 滚动峰值增长 | 0 / 6.18 MiB | 0 / 0.47 MiB |
| Chrome 内存基线 → 末窗口 | 不适用 | 694.44 → 280.09 MiB |
| Chrome 持续增长 / 滚动峰值增长 | 不适用 | 0 / 3.20 MiB |

全部 UI 样本中，排名 API 与脱敏 DTO 始终为 501 条，后台表格始终 8 行，大屏只出现正常分页的 12 行或末页 9 行，快照 ID 始终固定。快照超过 120 秒后出现过期提示属于已确认的新鲜度规则，不是 DOM 漂移。第三轮结果满足《开发文档》第 15.4 节和第 17 节的手动正式版稳定性门槛。
