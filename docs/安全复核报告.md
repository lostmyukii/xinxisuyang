# 安全复核报告

> 复核日期：2026-07-19
>
> 数据模式：本机手动导入；自动采集未启用
> 复核范围：本机 API、SQLite、管理后台、展示大屏、Excel 导出和 Chrome 只读探针

## 结论

当前手动导入模式满足本机运行的主要安全边界，可以进入合成数据验收。Chrome 扩展尚未证明能完整读取金山表格，因此只能用于结构探测，不能启用自动成绩同步。

## 检查结果

| 检查项 | 结果 | 证据与说明 |
|---|---|---|
| 本机监听 | 通过 | 服务拒绝非 `127.0.0.1` 的 `HOST`，后台和扩展均只访问环回地址 |
| 跨域限制 | 通过 | API 只接受明确配置的本机后台 Origin；不允许任意网页跨域调用 |
| 扩展最小权限 | 通过 | Manifest V3 只声明 KDocs 域名、本机 API、`activeTab` 和 `storage`；运行时还校验 `kdocs.cn` 和固定文档标识 `v130lGtJ` |
| 探针只读 | 通过 | 只统计 DOM 结构并发送哈希元数据，不点击、写入、截屏或读取 Cookie |
| 配对令牌 | 通过 | 探针上报必须携带本次本机服务的随机令牌；请求日志对令牌字段脱敏 |
| 外部输入校验 | 通过 | Zod 校验 API 请求；Fastify 请求体上限 15 MiB；文本、Base64、字段、规则数量另有限制 |
| 敏感列存储 | 通过 | 手机号和身份证号使用 AES-256-GCM 列加密；32 字节密钥保存在 macOS Keychain |
| 原子发布 | 通过 | SQLite 事务内写入完整快照并最后切换 current；规则重算、历史复用和恢复均有事务边界，失败回滚且保留上一成功快照 |
| 大屏最小 DTO | 通过 | 大屏接口白名单仅含赛区、赛项、组别、脱敏姓名、成绩和名次 |
| 导出隐私 | 通过 | 默认 Excel 不含手机号和身份证号；敏感导出要求固定二次确认文本并记录审计事件 |
| 临时文件 | 通过 | 导入和导出在内存中处理，不生成失败临时文件；稳定性测试使用系统临时目录并在退出时清理 |
| Git 忽略 | 通过 | 数据库、导出文件、HAR、日志、密钥、环境文件和测试报告均被忽略 |
| 合成测试数据 | 通过 | 仓库测试使用“虚构/合成/TEST”值，不保存真实选手信息 |
| 生产依赖审计 | 通过 | `pnpm audit --prod` 无已知漏洞；ExcelJS 的间接 `uuid` 已通过 workspace override 固定到修复版 11.1.1 |

## 攻击面与残余风险

1. 管理后台显示完整姓名，能登录比赛管理 Mac 的本地用户可见；比赛期间必须锁定电脑并限制物理访问。
2. 配对码会显示在启动终端中；不要截屏或转发，重启后默认自动更换。
3. Node.js 24 的 `node:sqlite` 当前会输出 ExperimentalWarning。功能测试已通过，但正式比赛前应固定 Node 24 的已验收小版本，不在现场临时升级。
4. 手工复制区域可能遗漏行。发布前必须核对金山页面总数与导入预览“读取记录”数量。
5. 自动采集仍是 `未完成`。在没有完整字段、完整记录、稳定 ID 和三次一致性证据前，不得把结构探针结果当作成绩同步。
6. 系统默认清理超过 30 天且非当前的历史快照，当前成功快照始终保留；这不等于完整赛事数据自动销毁。赛后总保留期限和删除数据库仍由赛事负责人决定，删除前必须确认备份与授权。

## 现场安全基线

- 只运行 `pnpm start:local`，不要修改 `HOST=127.0.0.1`。
- 不复制 Chrome 用户资料或金山登录态到项目目录。
- 不在聊天、Issue、Git 提交或测试夹具中粘贴真实成绩。
- 大屏使用 `/display`，不要把管理后台投到公众屏幕。
- 导出敏感字段前再次确认接收人、保存位置和保留期限。
- 结束后用 `Control-C` 停止服务，关闭金山标签页，并按赛事制度处理数据库和导出文件。
