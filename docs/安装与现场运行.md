# macOS 安装与现场运行

> 适用电脑：当前比赛管理 Mac
>
> 正式采集模式：手动导入
> 金山文档权限：只读，不修改裁判原表

## 1. 首次准备

在“终端”中进入项目目录：

```bash
cd /Users/yukii/Desktop/信息素养大赛成绩核对
pnpm check:environment
pnpm package:local
```

环境检查会核对 Node.js 24、pnpm 11、Google Chrome、macOS Keychain 与 `caffeinate` 命令、端口和本地数据目录。`package:local` 会安装锁定版本依赖，依次运行类型检查、代码检查、自动测试和构建，并生成：

- 管理后台：`apps/web/dist`
- 本机 API：`apps/server/dist`
- Chrome 只读探针：`apps/extension/dist`

首次启动时，macOS 可能询问是否允许访问钥匙串。允许后，SQLite 敏感列加密密钥由 Keychain 管理，不写入项目文件。

## 2. 日常启动与退出

启动：

```bash
cd /Users/yukii/Desktop/信息素养大赛成绩核对
pnpm start:local
```

看到“赛事指挥台”地址后，用 Chrome 打开：

```text
http://127.0.0.1:4173
```

本机服务固定监听 `127.0.0.1:4318`，同一网络中的其他电脑不能直接访问。结束比赛后回到终端按 `Control-C`，等待两个本地进程退出，再关闭浏览器页面。

开发调试使用 `pnpm dev:local`，比赛现场应使用构建后的 `pnpm start:local`。

## 3. 首次业务配置

1. 打开“赛项配置”。
2. 为每个赛项填写允许的最低分和最高分；系统不默认使用 0–100。
3. 成绩最多允许两位小数。
4. 保存后进入“手动导入”。
5. 先生成预览，核对读取数、有效数、异常数、排名分区和缺失规则。
6. 确认候选数据正确后，才点击“发布为当前快照”。
7. 在“同步管理”确认本次 `SyncRun` 为成功；如现场节奏需要，可调整“可能过期”和“严重过期”阈值，严重阈值必须更大。

发布失败不会覆盖上一成功快照。若需要核对或回到旧版本，可在“历史快照”查看内容、与当前版本比较后再选择恢复；恢复只切换当前版本，不修改历史内容。

## 4. 从金山文档导入

裁判继续在下列金山文档中填写：

```text
https://www.kdocs.cn/wo/sl/v130lGtJ
```

推荐现场流程：

1. Chrome 登录有权访问该表格的金山账号并保持页面打开。
2. 在金山表格中选中含表头的完整数据区域并复制。
3. 回到“手动导入”，选择“表格复制”并粘贴。
4. 核对字段映射。必填字段为赛区、赛项、组别、选手姓名和成绩；手机号、身份证号留空即可。
5. 生成预览。未配置赛项、空值、非数字、超过两位小数和超范围成绩会列为异常并排除排名。
6. 确认后发布。

也可使用 CSV 或 XLSX。文件只传给本机 `127.0.0.1` 解析，不上传到第三方服务。

## 5. 排名、大屏与导出

- “成绩排名”固定按“赛区＋赛项＋组别”分区，同分采用 `1、2、2、4`。
- “异常记录”只用于核对原因；裁判原始成绩应回到金山文档修正，再重新导入。
- 展示大屏地址为 `http://127.0.0.1:4173/display`。Chrome 可按 `Control-Command-F` 进入或退出全屏。
- 大屏每个排名分区每页最多显示 12 条，超过一页可手动翻页；开启“自动轮播”后会依次切换页和分区，不会静默截断记录。
- 大屏只取得脱敏姓名、赛区、赛项、组别、成绩和名次，不取得手机号或身份证号。
- “导出中心”支持当前分组、当前赛项和全部赛项。全部赛项下载为 ZIP，每个赛项一个 XLSX。
- 默认导出不含敏感字段；包含敏感字段必须进行二次确认，并只在确有授权时使用。

大屏和后台均显示最近成功时间及新鲜度。若页面过期、关闭或登录失效，继续保留上一成功排名，但现场人员必须根据新鲜度判断是否可公示。

## 6. Chrome 只读探针安装

当前扩展只用于验证金山页面结构，不能读取完整成绩，自动采集状态必须保持“未验证”。扩展还会校验当前页必须是 `kdocs.cn` 且文档标识为 `v130lGtJ`，不会对其他金山文档执行探针。安装步骤：

1. 先运行 `pnpm package:local`。
2. Chrome 打开 `chrome://extensions`。
3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择 `/Users/yukii/Desktop/信息素养大赛成绩核对/apps/extension/dist`。
6. 保持本机服务运行，从终端日志复制本次“Chrome 扩展配对码”。
7. 打开目标金山文档，点击扩展图标，保存配对码，再点击“执行只读结构检查”。
8. 只有页面提示探针已送达，且 `http://127.0.0.1:4318/api/probe/status` 有新结果，才算连接成功。

扩展权限仅包括目标 KDocs 域名、本机 API、当前标签页和扩展本地存储。它不读取 Cookie，不点击或改写单元格，不做 OCR/截图识别。每次未配置固定 `PAIRING_TOKEN` 时，本机服务重启会生成新的配对码，需要重新保存。

## 7. 现场故障处理

| 现象 | 处理 |
|---|---|
| 页面提示无法连接本机服务 | 确认启动终端仍在运行；重新执行 `pnpm start:local` |
| 端口已被占用 | 执行 `pnpm check:environment` 查看 4318/4173；先确认并退出旧的本系统进程 |
| 导入预览提示赛项未配置 | 先到“赛项配置”增加该赛项范围，再重新预览 |
| 发布失败 | 不要反复覆盖数据；确认上一成功快照仍可见，再检查字段和输入文件 |
| 金山登录失效或页面关闭 | 当前为手动模式，不影响最近成功快照；重新登录后再次复制导入 |
| 大屏显示数据过期 | 暂停公示新结果，重新导入并确认最近成功时间 |
| 扩展显示配对失败 | 从本次启动日志复制新配对码，确认本机 API 端口为 4318 |

## 8. 数据保留与清理

- 默认数据库：`/Users/yukii/Desktop/信息素养大赛成绩核对/data/competition.sqlite`
- 非当前历史快照默认保留最近 30 天；当前快照不因保留策略自动删除。相同内容再次发布不会创建重复快照，恢复后再次导入已存在的内容会复用原成功版本。
- 导出文件由 Chrome 保存到用户选择的下载目录，不写入 Git。
- 不要把 `data`、导出文件、`.env`、日志或 Chrome 用户数据提交到仓库。
- 30 天策略不等于赛事数据自动销毁；如需删除全部比赛数据，先停止服务并按赛事数据管理要求备份。删除数据库属于不可恢复操作，必须由负责人明确确认后执行。
- 不复制 Chrome 登录目录，不把金山账号、配对码或真实成绩写入问题报告。
